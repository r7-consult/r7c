# Plugin Manager (r7c)

## О проекте

**R7 Plugin Manager** — настольный менеджер плагинов для экосистемы R7/ONLYOFFICE.
Он облегчает жизнь администраторам и разработчикам, позволяя:

1. Просматривать каталог доступных плагинов с превью, описанием, changelog и рейтингами.
2. Устанавливать, обновлять и удалять плагины прямо из редактора без ручной сборки.
3. Загружать приветствие, лицензию и медиа-контент динамически из GitHub, чтобы быстро
   распространять обновления.
4. Предоставлять fallback-ссылки на готовые сборки (deploy), если автоматическая установка
   недоступна — пользователи всегда могут скачать нужный пакет вручную.
5. Управлять коммерческими плагинами: отображать карточку «Learn more», проверять доступ
   перед установкой и вести метрики кликов.

### Кому полезно
- **Команды поддержки** — чтобы оперативно выкатывать обновления и держать все плагины синхронизированными.
- **Разработчики** — получают единый storefront, где можно протестировать UI, доставку и телеметрию плагинов.
- **Конечные пользователи** — устанавливают плагины за пару кликов, не копаясь в GitHub и не перезапуская редактор.

## Структура репозитория

```
r7c/
 ├─ LICENSE.md
 ├─ README.md
 ├─ config.json
 ├─ developer.html
 ├─ index.html
 ├─ welcome.md
 ├─ store/
 │   ├─ scripts/
 │   ├─ resources/
 │   ├─ translations/
 │   └─ ... (плагины и storefront)
 ├─ resources/
 ├─ scripts/
 ├─ translations/
 ├─ vendor/
 └─ warning.html
```

Ключевые моменты:

1. Файлы `LICENSE.md`, `NOTICE` и `3rd-Party.txt` обеспечивают соблюдение требований Apache 2.0 и фиксируют происхождение кода.
2. Папка `store/` содержит интерфейс Plugin Manager (HTML/JS/CSS + контент), верхний уровень — служебные конфиги и оболочка.
3. Такая структура повторяет фактическую организацию проекта и применяется на практике в крупных гибридных (open+closed) решениях.
4. Проприетарные доработки описаны в `LICENSE.md`, поэтому новая логика может распространяться под закрытой лицензией, не нарушая условий Apache 2.0 для исходных компонентов.
